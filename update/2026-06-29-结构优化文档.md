## 变更详情

### 1. 源代码目录整合

**变更前：**
```
source/
├── cpp/                    # C++ 源代码
└── py/                    # Python 源代码

source-cpp/                # 重复的 C++ 源代码
├── *.cpp                  # 200+ 个重复文件
├── *.h
├── duipai/                # 对拍工具
├── .cph/                  # Competitive Companion 配置
└── .cph-ng/               # cph-ng 配置
```

**变更后：**
```
src/
├── cpp/                   # 统一的 C++ 源代码
│   ├── __DEBUG_TOOL.h     # 调试工具头文件
│   ├── *.cpp              # 所有 C++ 代码（合并自两个目录）
│   └── __DEBUG_TOOL.h
└── py/                    # Python 源代码
```

**说明：**
- 将 `source/cpp/` 和 `source-cpp/` 中的所有 `.cpp` 和 `.h` 文件合并到 `src/cpp/`
- 删除重复的 `__DEBUG_TOOL.h` 文件，保留 `src/cpp/__DEBUG_TOOL.h`
- Python 代码保持在 `src/py/`（原 `source/py/`）

**回滚命令：**
```powershell
# 创建 source 目录
New-Item -ItemType Directory -Force -Path "c:\Code\AlgorithmSolves\source\cpp","c:\Code\AlgorithmSolves\source\py"

# 从 src/cpp 移动 .cpp 和 .h 文件回 source/cpp
Get-ChildItem -Path "c:\Code\AlgorithmSolves\src\cpp" -Include "*.cpp","*.h" -Recurse | Move-Item -Destination "c:\Code\AlgorithmSolves\source\cpp" -Force

# 从 src/py 移动 .py 文件回 source/py
Get-ChildItem -Path "c:\Code\AlgorithmSolves\src\py" -Include "*.py" -Recurse | Move-Item -Destination "c:\Code\AlgorithmSolves\source\py" -Force

# 创建 source-cpp 目录并移动对拍工具
New-Item -ItemType Directory -Force -Path "c:\Code\AlgorithmSolves\source-cpp"
Move-Item -Path "c:\Code\AlgorithmSolves\tools\duipai" -Destination "c:\Code\AlgorithmSolves\source-cpp\" -Force
```

## 注意事项

1. **笔记图片路径**：笔记中的图片引用使用相对路径（如 `![](./image-1.png)`），移动后路径仍然有效，无需修改。

2. **脚本路径依赖**：更新了 `scripts/` 目录下的脚本路径，从 `source` 改为 `src`。如果使用 Competitive Companion 或 cph-ng 插件，请确保脚本路径配置正确。

3. **编译产物清理**：删除了所有 `.bin` 编译产物文件，这些文件可以重新生成。

4. **模板文件位置**：`templates/` 目录保持不变，所有模板文件位置未变。

5. **VS Code 配置**：`.vscode/settings.json` 中可能包含路径配置，请检查并更新（如有需要）。
