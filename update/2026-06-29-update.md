## 二、cph-ng 配置修复

### 2.1 根目录配置
- 在 `AlgorithmSolves/.vscode/settings.json` 创建了工作区配置
- 配置了 `cph-ng.companion.customPathScript` 指向自定义脚本

### 2.2 User Script 语言选择
**文件**: `scripts/cph-path-script.js`

选择语言时自动完成以下操作：
1. **弹出语言选择对话框**：选择 C++ 或 Python
2. **自动生成文件**：文件生成到 `src/cpp/` 或 `src/py/`
3. **返回路径数组**：cph-ng 根据返回的路径创建文件

### 2.3 模板修正机制
**文件**: `scripts/template-watcher.js`

作为安全网，自动修正模板：
- 监控 `src/cpp/` 和 `src/py/` 目录
- 新文件创建时，根据扩展名自动应用正确模板
- C++ 文件 → TEMPLATE-2.cpp
- Python 文件 → TEMPLATE-PY.py

### 2.4 自动启动配置
**文件**: `.vscode/tasks.json`

配置 template-watcher 在打开工作区时自动启动：
- 后台运行，不显示终端窗口
- `runOn: "folderOpen"` 确保每次打开 VS Code 自动启动

## 三、辅助工具

### 3.1 PowerShell 模板切换脚本
**文件**: `scripts/switch-template.ps1`

手动切换模板配置：
```powershell
.\scripts\switch-template.ps1 cpp    # 切换到 C++ 模板
.\scripts\switch-template.ps1 py     # 切换到 Python 模板
```

## 四、使用流程

### 爬取题目（一键完成）
```
浏览器爬取题目 → 选择语言（C++/Python）→ 自动完成所有操作
```

具体步骤：
1. 在 VS Code 中打开 `AlgorithmSolves` 根目录（watcher 自动启动）
2. 在浏览器点击 Competitive Companion 插件
3. 弹出选择框，选择语言（C++ / Python）
4. **全部自动完成**：
   - 文件生成到对应目录（`src/cpp/` 或 `src/py/`）
   - 应用正确模板内容
