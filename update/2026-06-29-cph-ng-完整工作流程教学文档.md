## 1. 数据传输流程

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    浏览器 (Browser)                              │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │           Competitive Companion 扩展                      │    │
│  │  1. 解析题目页面 → 2. 提取题目信息 → 3. 生成 JSON →        │    │
│  │  4. HTTP POST 请求发送到 localhost:10043                   │    │
│  └─────────────────────────────────────────────────────────┘    │
└──────────────────────────────────┬──────────────────────────────┘
                                   │ HTTP POST
                                   ↓
┌─────────────────────────────────────────────────────────────────┐
│                    VS Code (cph-ng 扩展)                        │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │              CompanionCommunicationService              │    │
│  │  1. 监听 localhost:10043 → 2. 接收 JSON →                │    │
│  │  3. 解析数据 → 4. 调用 UserScriptService                  │    │
│  └─────────────────────────────────────────────────────────┘    │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │                UserScriptService                        │    │
│  │  1. 读取 customPathScript → 2. 创建 sandbox →            │    │
│  │  3. 执行 process() → 4. 获取返回路径数组 →                │    │
│  │  5. 创建问题文件                                         │    │
│  └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 端口配置

cph-ng 默认监听端口 `10043`，可通过以下配置修改：

```json
"cph-ng.companion.listenPort": 10043
```

## 2. Competitive Companion JSON 数据格式

### 2.1 单题数据格式

当你在浏览器点击 Competitive Companion 扩展图标时，发送的数据格式如下：

```json
{
    "name": "A. Problem Name",
    "url": "https://codeforces.com/contest/1234/problem/A",
    "group": "Codeforces Round #580 (Div. 2)",
    "interactive": false,
    "memoryLimit": 256,
    "timeLimit": 2000,
    "tests": [
        {
            "input": "3\n1 2 3",
            "output": "6"
        },
        {
            "input": "5\n5 4 3 2 1",
            "output": "15"
        }
    ],
    "src": "https://codeforces.com/contest/1234/problem/A",
    "type": "Codeforces",
    "tags": ["math", "implementation"],
    "source": "Codeforces"
}
```

### 2.2 多题数据格式（比赛页面）

当你在比赛页面点击扩展图标时，会发送一个数组：

```json
[
    {
        "name": "A. Problem A",
        "url": "https://codeforces.com/contest/1234/problem/A",
        "tests": [...]
    },
    {
        "name": "B. Problem B",
        "url": "https://codeforces.com/contest/1234/problem/B",
        "tests": [...]
    },
    ...
]
```

### 2.3 关键字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 题目名称（如 "A. Problem Name"） |
| `url` | string | 题目 URL |
| `group` | string | 比赛名称（如 "Codeforces Round #580"） |
| `memoryLimit` | number | 内存限制（MB） |
| `timeLimit` | number | 时间限制（ms） |
| `tests` | array | 测试用例数组 |
| `tests[].input` | string | 输入数据 |
| `tests[].output` | string | 输出数据 |
| `interactive` | boolean | 是否为交互式题目 |
| `type` | string | 平台类型（Codeforces/AtCoder/Luogu 等） |

## 3. cph-ng User Script API

### 3.1 概述

cph-ng 提供了一个 User Script API，允许用户自定义文件路径生成逻辑。脚本在 Node.js 的 sandbox 环境中执行，只提供有限的全局变量。

### 3.2 必须实现的函数

脚本必须导出一个 `async function process()` 函数，作为入口点：

```javascript
async function process() {
    // 你的逻辑
    return ['/path/to/file1.cpp', '/path/to/file2.cpp'];
}
```

### 3.3 全局变量

#### 3.3.1 `problems` - 题目数据数组

```typescript
interface CompanionProblem {
    name: string;
    url: string;
    group: string;
    interactive: boolean;
    memoryLimit: number;
    timeLimit: number;
    tests: Array<{
        input: string;
        output: string;
    }>;
    src: string;
    type: string;
    tags: string[];
    source: string;
}

// 在脚本中可直接使用
const problems: CompanionProblem[];
```

**使用示例**：

```javascript
for (const problem of problems) {
    console.log(problem.name);      // "A. Problem Name"
    console.log(problem.group);     // "Codeforces Round #580"
    console.log(problem.url);       // "https://codeforces.com/contest/1234/problem/A"
    console.log(problem.tests.length); // 测试用例数量
}
```

#### 3.3.2 `workspaceFolders` - 工作区文件夹数组

```typescript
interface WorkspaceFolderContext {
    path: string;      // 注意：是 .path，不是 .uri.fsPath
}

// 在脚本中可直接使用
const workspaceFolders: WorkspaceFolderContext[];
```

**使用示例**：

```javascript
// 获取第一个工作区的根目录路径
const root = workspaceFolders[0].path;
// 返回: "C:\\Code\\AlgorithmSolves"
```

#### 3.3.3 `path` - Node.js path 模块

提供完整的 Node.js `path` 模块：

```javascript
const targetDir = path.join(root, 'src', 'cpp');
// 返回: "C:\\Code\\AlgorithmSolves\\src\\cpp"

const ext = path.extname('main.cpp');
// 返回: ".cpp"

const basename = path.basename('src/cpp/main.cpp', '.cpp');
// 返回: "main"
```

#### 3.3.4 `fs` - 受限的文件系统模块

**注意**：sandbox 只提供 `fs.existsSync()`，不提供其他文件操作函数！

```javascript
// 只能检查文件/目录是否存在
if (fs.existsSync(targetDir)) {
    console.log('目录已存在');
}

// ❌ 以下函数不可用：
// fs.mkdirSync()
// fs.readFileSync()
// fs.writeFileSync()
// fs.readdirSync()
// 等
```

#### 3.3.5 `ui` - 用户界面交互模块

提供与 VS Code UI 交互的函数。**注意**：根据 cph-ng 官方文档，目前只明确支持 `chooseItem` 方法：

```typescript
interface UI {
    // 显示选择对话框，返回选中的字符串或 undefined（取消）
    // items: 字符串数组，placeHolder: 对话框占位文本（可选）
    chooseItem(items: string[], placeHolder?: string): Promise<string | undefined>;
}

// 在脚本中可直接使用
const ui: UI;
```

**使用示例**：

```javascript
// 选择语言
const lang = await ui.chooseItem(['C++', 'Python'], 'Choose language');
if (!lang) return 'Cancelled';
```

**注意**：其他 UI 方法（如 `showInformationMessage`）在 cph-ng sandbox 中可能不可用。如需调试，建议使用返回错误消息的方式（详见第 7 节）。

### 3.4 返回值类型

`process()` 函数必须返回以下类型之一：

| 返回类型 | 说明 |
|----------|------|
| `(string \| null)[]` | 文件路径数组，每个元素对应 problems 数组中的一个题目 |
| `string` | 错误消息，会显示给用户并中止导入 |

**返回值约束**：
1. **长度必须匹配**：数组长度必须等于 `problems.length`
2. **必须是绝对路径**：每个字符串必须是绝对路径
3. **只能是 string 或 null**：不能返回其他类型

**正确示例**：

```javascript
async function process() {
    const root = workspaceFolders[0].path;
    const results = [];
    
    for (const problem of problems) {
        const name = problem.name.replace(/[\\/:*?"<>|]/g, '_');
        const filePath = path.join(root, 'src', 'cpp', name + '.cpp');
        results.push(filePath);
    }
    
    return results;
}
```

**错误示例**：

```javascript
// ❌ 返回相对路径（错误）
return ['./src/cpp/main.cpp'];

// ❌ 返回数组长度不匹配（错误）
return ['file1.cpp']; // 但 problems 有 3 个元素

// ❌ 返回对象（错误）
return [{ path: 'file.cpp' }];
```

## 4. 自定义脚本实现详解

### 4.1 脚本完整代码

```javascript
async function process() {
    // 步骤 1：弹出语言选择对话框
    const lang = await ui.chooseItem(
        ['C++', 'Python'],
        'Choose language'
    );

    // 步骤 2：处理取消操作
    if (!lang) return 'Cancelled';

    // 步骤 3：根据选择确定扩展名和目标目录
    const isCpp = lang === 'C++';
    const ext = isCpp ? '.cpp' : '.py';

    // 步骤 4：获取工作区根目录
    const root = workspaceFolders[0].path;
    const targetDir = path.join(root, 'src', isCpp ? 'cpp' : 'py');

    // 步骤 5：为每个题目生成文件路径
    const results = [];
    for (const problem of problems) {
        // 清理题目名称中的非法字符
        const name = problem.name.replace(/[\\/:*?"<>|]/g, '_');
        const filePath = path.join(targetDir, name + ext);
        results.push(filePath);
    }

    // 步骤 6：返回路径数组
    return results;
}
```

### 4.2 关键实现细节

#### 4.2.1 语言选择

```javascript
const lang = await ui.chooseItem(
    ['C++', 'Python'],  // ✅ 必须是字符串数组，不能是对象数组
    'Choose language'   // 可选：对话框占位文本
);
```

**为什么用字符串数组？**

根据 cph-ng 官方 API 定义：
```typescript
chooseItem(items: string[], placeHolder?: string): Promise<string | undefined>;
```

如果传入对象数组 `[{ label: 'C++' }]`，对话框会显示为空，点击无反应，最终返回 `undefined`，导致 "Cancelled"。

#### 4.2.2 工作区路径获取

```javascript
const root = workspaceFolders[0].path;  // ✅ 正确
// const root = workspaceFolders[0].uri.fsPath;  // ❌ 错误
```

**为什么用 `.path`？**

根据 cph-ng 的 `WorkspaceFolderContext` 接口定义：
```typescript
interface WorkspaceFolderContext {
    path: string;
}
```

`.uri.fsPath` 是 VS Code 原生 API 的属性，但 cph-ng 的 sandbox 环境只提供了 `.path`。

#### 4.2.3 题目名称清理

```javascript
const name = problem.name.replace(/[\\/:*?"<>|]/g, '_');
```

**为什么需要清理？**

题目名称可能包含文件系统不允许的字符，如：
- Windows 不允许：`\ / : * ? " < > |`
- 示例：题目名称 "A/B Problem" 会被转换为 "A_B Problem"

#### 4.2.4 目标目录结构

```javascript
const targetDir = path.join(root, 'src', isCpp ? 'cpp' : 'py');
// C++ → "C:\Code\AlgorithmSolves\src\cpp"
// Python → "C:\Code\AlgorithmSolves\src\py"
```

**为什么这样设计？**

- 统一源代码入口：`src/` 目录
- 按语言分类：`cpp/` 和 `py/` 子目录
- 便于查找和管理：所有 C++ 代码在一个目录，所有 Python 代码在另一个目录

## 5. 问题与解决方案

### 5.1 问题列表

| 问题编号 | 问题描述 | 原因 | 解决方案 |
|----------|----------|------|----------|
| P01 | 语言选择对话框为空，点击无反应 | `ui.chooseItem` 参数类型错误（对象数组） | 改为字符串数组 `['C++', 'Python']` |
| P02 | 脚本无法获取正确的工作区路径 | 使用了 `.uri.fsPath`，sandbox 不提供 | 改为 `.path` |
| P03 | Python 文件创建时应用 C++ 模板 | `cph-ng.problem.templateFile` 只能设置一个模板 | 使用 `template-watcher.js` 自动修正 |
| P04 | 脚本无法读取模板文件 | sandbox 只提供 `fs.existsSync`，无 `fs.readFileSync` | 由 cph-ng 负责文件创建和模板应用 |
| P05 | template-watcher 需要手动启动 | Node.js 脚本需要手动运行 | 配置 `tasks.json` 自动启动 |
| P06 | 题目名称包含非法字符 | 文件系统不允许某些字符 | 使用正则表达式替换非法字符 |

### 5.2 详细解决方案

#### P01：语言选择对话框为空

**错误代码**：
```javascript
// ❌ 错误
const lang = await ui.chooseItem([
    { label: 'C++', description: 'C++ 语言' },
    { label: 'Python', description: 'Python 语言' }
]);
```

**正确代码**：
```javascript
// ✅ 正确
const lang = await ui.chooseItem(
    ['C++', 'Python'],
    'Choose language'
);
```

**原因分析**：cph-ng 的 `ui.chooseItem` API 只接受 `string[]` 类型，传入对象数组会被忽略，导致对话框显示为空。

#### P02：工作区路径获取错误

**错误代码**：
```javascript
// ❌ 错误
const root = workspaceFolders[0].uri.fsPath;
```

**正确代码**：
```javascript
// ✅ 正确
const root = workspaceFolders[0].path;
```

**原因分析**：cph-ng 在 sandbox 中封装了 `WorkspaceFolderContext`，只暴露了 `.path` 属性，没有暴露完整的 VS Code URI 对象。

#### P03：Python 文件应用错误模板

**问题**：`cph-ng.problem.templateFile` 只能配置一个模板文件：

```json
"cph-ng.problem.templateFile": "C:\\Code\\AlgorithmSolves\\templates\\TEMPLATE-2.cpp"
```

**解决方案**：使用 `template-watcher.js` 作为安全网：

```javascript
function applyTemplate(filePath) {
    const ext = path.extname(filePath);
    
    if (ext === '.cpp') {
        // 保持 C++ 模板（已正确）
    } else if (ext === '.py') {
        // 覆盖为 Python 模板
        fs.writeFileSync(filePath, pyTemplate);
    }
}

function watchDir(dir) {
    fs.watch(dir, (eventType, filename) => {
        if (eventType === 'rename' && filename) {
            const filePath = path.join(dir, filename);
            // ⚠️ 关键：延迟 100ms 后再读取文件
            setTimeout(() => {
                if (fs.existsSync(filePath)) {
                    applyTemplate(filePath);
                }
            }, 100);
        }
    });
}
```

**工作原理**：
1. cph-ng 创建文件时，默认应用 `TEMPLATE-2.cpp` 的内容
2. `template-watcher.js` 监控文件创建事件
3. 检测到 `.py` 文件时，立即覆盖为 `TEMPLATE-PY.py` 的内容
4. 检测到 `.cpp` 文件时，保持不变

**为什么需要 setTimeout(100ms)？**

这是为了解决**竞态条件**（Race Condition）问题：

- `fs.watch()` 在文件被创建时立即触发回调，但此时 cph-ng 可能还在写入文件内容
- 如果 watcher 立即读取文件，可能会读到不完整的内容（部分写入状态）
- 延迟 100ms 确保 cph-ng 完成文件写入后，watcher 再读取和修改文件
- 这个时间间隔足够处理大多数情况，不会影响用户体验

#### P04：脚本无法读取模板文件

**问题**：sandbox 环境限制，无法使用 `fs.readFileSync`：

```javascript
// ❌ 错误：在 sandbox 中不可用
const fs = require('fs');
const templateContent = fs.readFileSync(templatePath, 'utf-8');
```

**解决方案**：让 cph-ng 负责文件创建和模板应用：

1. 脚本只返回文件路径数组
2. cph-ng 根据返回的路径创建文件
3. cph-ng 使用配置的 `templateFile` 填充文件内容
4. `template-watcher.js` 修正 Python 文件的模板

**为什么这样设计？**

- cph-ng 的 sandbox 设计目的是安全隔离，防止脚本执行恶意操作
- 文件操作由 cph-ng 自己处理，脚本只负责路径生成
- 这种设计符合 cph-ng 的架构理念

#### P05：template-watcher 需要手动启动

**解决方案**：配置 `.vscode/tasks.json`：

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Start Template Watcher",
            "type": "shell",
            "command": "node",
            "args": ["${workspaceFolder}/scripts/template-watcher.js"],
            "isBackground": true,
            "presentation": {
                "reveal": "never",
                "panel": "dedicated"
            },
            "runOptions": {
                "runOn": "folderOpen"
            }
        }
    ]
}
```

**关键配置说明**：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `isBackground` | `true` | 后台运行，不阻塞其他操作 |
| `reveal` | `"never"` | 不自动显示终端窗口 |
| `panel` | `"dedicated"` | 使用专用终端面板 |
| `runOn` | `"folderOpen"` | 打开工作区时自动运行 |

#### P06：题目名称包含非法字符

**解决方案**：使用正则表达式替换：

```javascript
const name = problem.name.replace(/[\\/:*?"<>|]/g, '_');
```

**替换规则**：

| 非法字符 | 替换为 |
|----------|--------|
| `\` | `_` |
| `/` | `_` |
| `:` | `_` |
| `*` | `_` |
| `?` | `_` |
| `"` | `_` |
| `<` | `_` |
| `>` | `_` |
| `\|` | `_` |

## 6. 完整工作流程图

### 6.1 启动阶段

```
打开 VS Code
    ↓
加载工作区配置 (.vscode/tasks.json)
    ↓
自动启动 template-watcher.js
    ↓
watcher 开始监控 src/cpp/ 和 src/py/ 目录
    ↓
等待 Competitive Companion 发送题目数据
```

### 6.2 导入阶段

```
Competitive Companion 发送 JSON 数据
    ↓
cph-ng 接收数据并解析
    ↓
调用 UserScriptService.resolvePaths()
    ↓
读取 cph-path-script.js
    ↓
创建 sandbox 环境（注入全局变量：problems, workspaceFolders, path, fs, ui）
    ↓
执行 process() 函数
    ↓
弹出语言选择对话框
    ↓
用户选择语言（C++ / Python）
    ↓
生成文件路径数组
    ↓
返回路径数组给 cph-ng
    ↓
cph-ng 验证路径数组（长度匹配、绝对路径）
    ↓
cph-ng 创建问题文件（应用 TEMPLATE-2.cpp 模板）
    ↓
cph-ng 创建 .cph-ng/ 目录下的内部数据文件（.bin）
    ↓
template-watcher.js 检测到新文件创建
    ↓
根据文件扩展名应用正确模板：
    - .cpp 文件：保持 C++ 模板
    - .py 文件：覆盖为 Python 模板
    ↓
完成！
```

### 6.3 数据流向图

```
JSON 数据 (Competitive Companion)
    ↓
problems 全局变量 (sandbox)
    ↓
for 循环遍历每个题目
    ↓
生成绝对文件路径
    ↓
返回路径数组
    ↓
cph-ng 创建文件
    ↓
template-watcher.js 修正模板
    ↓
最终文件 (src/cpp/*.cpp 或 src/py/*.py)
```

### 6.4 .cph-ng/ 目录与内部数据文件

cph-ng 在创建题目文件的同时，还会在 `.cph-ng/` 目录下创建内部数据文件（`.bin` 文件），这是完整数据流中不可跳过的一环。

#### 目录结构

```
.cph-ng/
├── cpp/                    # 对应 src/cpp/ 中的题目
│   └── A._Problem_Name.bin # 存储题目元数据和测试用例
└── py/                     # 对应 src/py/ 中的题目
    └── B._Problem_Name.bin
```

#### .bin 文件的作用

| 作用 | 说明 |
|------|------|
| 存储测试用例 | 包含 Competitive Companion 发送的所有测试用例（输入/输出） |
| 存储题目元数据 | 包含题目名称、URL、时间限制、内存限制等信息 |
| 支持运行测试 | cph-ng 执行代码时，读取 `.bin` 文件中的测试用例进行验证 |
| 支持提交代码 | cph-ng 提交代码时，读取 `.bin` 文件中的题目信息定位到正确的提交页面 |

#### 配置项

`.bin` 文件的存储路径由以下配置项控制：

```json
"cph-ng.problem.problemFilePath": "${workspace}/.cph-ng/${relativeDirname}/${basename}.bin"
```

**变量说明**：

| 变量 | 说明 | 示例 |
|------|------|------|
| `${workspace}` | 工作区根目录 | `C:\Code\AlgorithmSolves` |
| `${relativeDirname}` | 源代码文件的相对目录 | `cpp` |
| `${basename}` | 源代码文件的基础名称（不含扩展名） | `A._Problem_Name` |

**生成的完整路径示例**：

```
C:\Code\AlgorithmSolves\.cph-ng\cpp\A._Problem_Name.bin
C:\Code\AlgorithmSolves\.cph-ng\py\B._Problem_Name.bin
```

#### 为什么需要 `.bin` 文件？

- **分离关注点**：源代码文件只包含代码，测试用例和元数据存储在单独的 `.bin` 文件中
- **避免污染代码**：测试用例数据不会出现在源代码中
- **便于管理**：可以独立删除或修改测试用例，不影响源代码
- **支持离线使用**：即使没有网络连接，也可以使用已保存的测试用例运行代码

## 7. 调试技巧

### 7.1 脚本调试

由于脚本在 sandbox 中执行，无法直接使用 `console.log()` 调试，且 `ui.showInformationMessage()` 等方法在 cph-ng sandbox 中可能不可用。以下是可靠的替代方案：

#### 方案 1：返回错误消息（推荐）

这是最可靠的调试方法。通过返回错误消息，可以在 VS Code 中看到调试信息：

```javascript
async function process() {
    // 调试：检查 problems 数组
    if (!problems || problems.length === 0) {
        return `错误：problems 数组为空`;
    }
    
    // 调试：检查工作区路径
    if (!workspaceFolders || workspaceFolders.length === 0) {
        return `错误：workspaceFolders 数组为空`;
    }
    
    // 调试：输出关键信息
    const root = workspaceFolders[0].path;
    return `调试信息：工作区路径=${root}，题目数量=${problems.length}`;
    
    // ... 正常逻辑
}
```

**工作原理**：当 `process()` 返回非数组字符串时，cph-ng 会将其作为错误消息显示给用户。

#### 方案 2：测试脚本逻辑（推荐）

创建一个测试文件 `test-script.js`，在本地环境中运行调试：

```javascript
// 模拟 sandbox 环境
const path = require('path');

// 模拟全局变量
const problems = [
    { name: 'A. Test Problem', url: 'https://example.com/problem/A' },
    { name: 'B. Another Problem', url: 'https://example.com/problem/B' }
];
const workspaceFolders = [{ path: '/path/to/workspace' }];
const ui = {
    chooseItem: async (items) => items[0], // 默认选择第一个
};
const fs = { existsSync: () => true };

// 导入你的脚本逻辑
async function process() {
    const lang = await ui.chooseItem(['C++', 'Python'], 'Choose language');
    if (!lang) return 'Cancelled';
    
    const isCpp = lang === 'C++';
    const ext = isCpp ? '.cpp' : '.py';
    const root = workspaceFolders[0].path;
    const targetDir = path.join(root, 'src', isCpp ? 'cpp' : 'py');
    
    const results = [];
    for (const problem of problems) {
        const name = problem.name.replace(/[\\/:*?"<>|]/g, '_');
        const filePath = path.join(targetDir, name + ext);
        results.push(filePath);
    }
    
    return results;
}

// 执行测试
process().then(console.log).catch(console.error);
```

运行测试：
```bash
node test-script.js
```

**优点**：
- 可以使用 `console.log()` 自由调试
- 可以模拟各种场景（空数组、特殊字符等）
- 无需依赖 cph-ng 扩展

### 7.2 cph-ng 日志

cph-ng 在 VS Code 的输出面板中提供日志：

1. 打开 VS Code 的输出面板（`Ctrl+Shift+U`）
2. 在下拉菜单中选择 "cph-ng"
3. 可以看到详细的日志信息，包括：
   - 接收的题目数据
   - 脚本执行结果
   - 文件创建情况
   - 错误信息

### 7.3 常见错误排查

| 错误信息 | 可能原因 | 排查方法 |
|----------|----------|----------|
| "Cancelled - 用户脚本未返回数组" | `ui.chooseItem` 参数错误或用户取消 | 检查参数是否为字符串数组 |
| "用户脚本执行失败" | 脚本语法错误 | 在本地测试脚本语法 |
| "返回路径数组长度不匹配" | `problems.length` 与返回数组长度不一致 | 添加调试信息检查长度 |
| "返回路径不是绝对路径" | 返回了相对路径 | 确保使用 `path.join(root, ...)` |
| "文件创建失败" | 目标目录不存在 | 确保目录已创建或让 cph-ng 自动创建 |

## 总结

### 核心要点

1. **数据流程**：Competitive Companion → HTTP POST → cph-ng → User Script → 文件创建
2. **脚本职责**：只负责返回文件路径数组，不负责文件创建和模板应用
3. **sandbox 限制**：只能使用 `problems`, `workspaceFolders`, `path`, `fs.existsSync`, `ui`
4. **模板处理**：cph-ng 应用默认模板，`template-watcher.js` 修正 Python 模板
5. **自动启动**：`tasks.json` 配置 watcher 在打开工作区时自动启动

### 常见陷阱

- ❌ `ui.chooseItem` 传入对象数组
- ❌ 使用 `workspaceFolders[0].uri.fsPath`
- ❌ 在脚本中使用 `fs.readFileSync` 或 `fs.writeFileSync`
- ❌ 返回相对路径或数组长度不匹配

### 推荐实践

- ✅ 使用 `path.join()` 构建绝对路径
- ✅ 清理题目名称中的非法字符
- ✅ 使用返回错误消息的方式调试（或在本地测试脚本）
- ✅ 在本地测试脚本逻辑后再部署
