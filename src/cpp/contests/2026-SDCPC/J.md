统一规约:边界 = 位置之间的"缝隙"
用整数 b 表示 divider 卡在 位置 b 和 b+1 之间(即 b 的右侧缝)。数轴:
缝隙:  0   1   2   3   4   5     <- 边界 b 取值
位置:    1   2   3   4   5       <- 松针位置
- b = 0:divider 在最左(所有针都在右边 → 全在盒子 A),这就是初始态。
- b = 5:在最右。
- b = 2:卡在位置 2 和 3 之间。
T:跨越集合是半开半闭 (lo, hi]
divider 从缝 lo 移到缝 hi,跨过的位置是 lo+1, lo+2, …, hi,也就是闭区间 [lo+1, hi],共 hi-lo 个:
return (hi - lo) - del(lo+1, hi) - pc(lo+1, hi);
//      ^^^^^^^ 跨过位置数    ^^^^^^^^^^^ 都用 [lo+1, hi]
起点缝 lo 那一格不含(你已经在它右侧了),终点缝 hi 左边那格含(刚跨过去)。所以是 (lo, hi]。三个量口径必须一致,全用 [lo+1, hi]。
sweep:覆盖集合是闭区间 [L, R]
int sweep = (cr - cl + 1) - del(cl, cr) - k;
//           ^^^^^^^^^^^^ 位置 L..R 全含   ^^^^^^^^^^ 都用 [cl, cr]
因为扫描要把 [L,R] 里的针全部插入,端点 L、R 本身也是本批针要插掉,所以是闭区间,减掉的 k 也含两端。sweep 最终 = [L,R] 内的挡路针数。
端点边界:停 R 用 R,停 L 用 L-1
由规约直接推:
状态	divider 卡在哪	边界值
停在 R(刚从左扫到右,插完 R)	R 的右侧缝	R(= cr)
停在 L(刚从右扫到左,插完 L)	L 的左侧缝	L-1(= cl-1)
这也解释了 T 的第二个参数为什么一个减 1、一个不减:
- 停 R → 先奔到 L 的左侧准备向右扫 → 目标缝 cl-1,T(pb, cl-1),接着 sweep 第一步就跨过位置 L。
- 停 L → 先奔到 R 的右侧准备向左扫 → 目标缝 cr,T(pb, cr),接着 sweep 第一步跨过位置 R。
衔接为什么不漏不重
- T(pb, cl-1) 覆盖到缝 cl-1 为止;sweep 从位置 cl 起。两段在位置上首尾相接、不缺格。
- 若上批终点 pb 落在 [L,R] 内或右侧,T 会把 [cl, cr] 也跨一遍、sweep 再扫一遍 → 这段被算两次,这正是物理上"先往左挪针、再往右扫回来"的双重经过,是对的(对拍已验证)。
- 本批针在两段各自被扣除(T 里 -pc,sweep 里 -k),因为本批针任何阶段都是"插入"不是"移动",两边都不让它计入移动代价,不会冲突。
一句话记忆:T 是缝到缝、半开半闭 (lo,hi];sweep 是位置到位置、闭区间 [L,R];停 R 落在 R 缝、停 L 落在 L-1 缝。 三者口径由"边界=右侧缝"这一条统一推出,所以不会串。
