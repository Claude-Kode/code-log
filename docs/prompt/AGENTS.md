# AGENTS.md

> 给 AI 助手的前置指令。此项目的使用者是竞赛编程背景的 C++ 选手，正在从"单文件脚本"向"工程开发"过渡。

## 使用者画像

- 竞赛编程导向的 C++ 选手（CF/AtCoder/校赛），算法功底扎实
- 数据结构（线段树/主席树/DSU/BIT/Trie/图论）、STL 选型、编译四阶段、g++ 命令行均熟练
- 工程开发零经验：没写过 CMake 项目，没做过多文件项目
- 同时在学习 Python（初学者）和 PowerShell（入门期）
- 学习风格：好奇心驱动，追概念到本质，AI 写的东西学完后能讲出原理才算自己的

## 交流规则

1. 始终使用简体中文回复
2. 代码注释和文档使用中文
3. 代码标识符和字符串内容保持原样
4. 不确定用户是否真会某知识点时，直接问"这你真会还是 AI 写的？"
5. 给建议时优先推荐"先跑起来一个小项目"的动手方案，降低心理门槛

## C++ 能力边界

**真会：**
- STL 容器选型、算法模板（见 `templates/TEMPLATE-FUNCTIONS.md` 全部内容）
- g++ 编译命令、freopen + 对拍 + #ifdef LOCAL 调试
- 静态库 vs 动态库、GOT/PLT 机制
- auto/结构化绑定/lambda/range-for/运算符重载/简单函数模板

**不会/不熟：**
- CMake / 多文件项目组织（概念懂了，没实践过）
- 智能指针/RAII/异常/多线程/网络编程（完全没接触）
- 模板元编程高级内容（SFINAE/if constexpr）
- 代码优先单文件、最小可运行

## Qt 学习约定

- 当前 Qt 水平：完全初学者
- 使用 Qt 6（不碰 Qt 5）
- 构建系统明确用 CMake（不用 qmake）
- 从 Qt Widgets 开始学（不用 Qt Quick/QML）
- 教学风格：先讲"这个类负责什么、怎么用"，再给最小可运行示例
- 每个新概念配一个独立单文件 demo，逐步过渡到多文件项目
- 遇到 CMake 配置时，逐行解释每个指令的含义
- 优先教信号与槽（Qt 核心机制）、控件布局、事件处理这三项基础

## 项目结构

Qt 练习代码放在 `practice/` 下，按主题分子目录（如 `signals-slots/`、`layouts/`、`widgets/`）。
