I — 树（武汉邀请赛）
2026 ICPC Wuhan Invitational  |  C++  |  93 行

解题思路

该题包含两个独立的子问题，通过输入的首行数据判断走哪个分支：

  • 子问题 1：给定一棵树，目标是选出 n-3 个节点（即删除 3 个度数最小的节点）。做法：找到任意一个叶子节点（度数为 1），从其唯一邻接点出发 BFS 遍历全树，输出 BFS 序的前 n-3 个节点。正确性基于叶子节点的邻接点总能作为 BFS 起点覆盖大部分节点。
  • 子问题 2：给定节点数 n 和一个序列 a[2..m+1]，要求根据度数约束重构一棵树。a[i] 编码了某种类似深度/度数前缀的信息：节点 2 的度数 = a[2]-1，后续节点 i 的度数 = a[i]-a[i-1]+1。构造时使用贪心：维护一个当前度数未满的节点指针 cur，每次给新节点 i 连一条边到 cur 上，直到 cur 的度数耗尽再移动 cur。

K — Permutation
交互题  |  C++  |  107 行

解题思路

这是一道交互题，需要通过与评测机对话恢复长度为 n 的隐藏排列。

  • 核心交互函数 ask(mid, x, y)：构造一个序列——前 mid 个位置全填 x，剩余位置全填 y，将序列发送给评测机。返回值表示 x 和 y 在此排列中的相对顺序关系（1 表示 x 在前面 / 不变，2 表示需要交换位置，0 表示无关或相等）。
  • 随机化分治策略：将当前区间 [l, r] 的候选值集合随机打乱，然后两两比较相邻的值。根据 ask 的返回值将每个值分配到左半区 [l, mid] 或右半区 [mid+1, r] 中，最后递归处理两个子区间。若有多余的值（配对剩余），根据左右半区已分配的数量决定其归属。
  • 关键技巧：使用 shuffle 随机打乱 + 逐对比较以避免最坏情况，同时保证分治规则的一致性。最终通过 1 次最终输出报告完整排列。

H — 矩形切割（武汉邀请赛）
2026 ICPC Wuhan Invitational  |  C++  |  79 行

解题思路

给定 n × m 的矩形，进行 q 次切割操作。每次操作指定方向（横向 op=1 或纵向 op=2）和切割位置 k，切割后输出当前所有子矩形中的最大面积。

  • 数据结构设计：维护四个 map——len / wid 记录每个切割线为起点所能得到的区间长度（key = 起始坐标, value = 长度），mx_len / mx_wid 按降序维护每种长度出现的次数（key = 长度, value = 计数）。
  • 切割操作：在位置 k 处切割时，用 upper_bound 找到 k 所在的区间 [x1, x2]，将其分裂为 [x1, k] 和 [k, x2]。更新 len/wid 的 value，并维护 mx_len/mx_wid 中的计数——旧长度减 1（消 0 删除），两个新长度各加 1。
  • 查询答案：每次操作后，最大面积 = 当前最大横向长度 × 最大纵向长度，即 mx_len.begin()->first * mx_wid.begin()->first。

复杂度分析

时间: O(q log q)   空间: O(q)

J — 排列子数组相似性 (105423)
105423  |  C++  |  88 行

解题思路

给定两个长度为 n 的排列 A 和 B，求有多少个区间 [l, r] 满足 A[l..r] 和 B[l..r] 包含完全相同的元素集合（顺序可以不同）。

  • 随机哈希映射：为每个数值 x (1..n) 分配一个随机的 64 位哈希值 val[x]，那么两个集合相等当且仅当它们的哈希值集合一致。
  • 线段树维护哈希：建立两棵线段树，分别对应排列 A 和排列 B。线段树按值的位置下标存储该值的哈希值。每个节点的哈希值 = 左子树哈希 + 右子树哈希 × base^(左子树元素个数)。这样两棵树根节点的哈希值相等就代表两个排列中相同位置集合的元素完全一致。
  • 双指针滑动窗口：固定右端点 r，不断向右扩展。当 [l, r] 内的两棵树哈希值不相等时，收缩左边界 l++（将 l 位置的值从线段树中移除）。当哈希值相等时，对答案贡献 r - l + 1。

复杂度分析

时间: O(n log n)   空间: O(n)
