1. 脚本指的是, 跟 cpp 不同, 不需要单独编译成二进制 文件后才能运行的编程语言
非常经典的实现方式就是解释器
解释器的特点就是 边读边运行, 一行一行读, 一行一行运行
一个是按行运行, 一个是按文件运行

1. 解释器 vs 编译器（本质区别）
•   编译型 (C++)：源码 → 机器码 → CPU 直跑。（快，静态）

•   解释型 (CPython)：源码 → 字节码 → VM 解释执行。（慢，动态）

•   关键：CPU 只认机器码。VM（解释器）是字节码与硬件之间的翻译官。

2. 既然有字节码，为何还要解释器？
•   跨平台：一份字节码，靠不同平台的 VM（解释器）适配硬件（Write Once, Run Anywhere）。

•   动态性：eval、反射等运行时修改代码的能力，必须在 VM 管控下才能实现，静态编译无法做到。

•   开发效率：牺牲运行时性能，换取秒级反馈和语法灵活性。

3. Python 底层是 C，怎么跑的？
•   CPython：Python 代码 → 字节码 → C 写的虚拟机（VM） 读取字节码 → 调用 C 函数 → 间接生成机器码。

•   你写的 Python 逻辑，最终是靠 C 在干活。

4. “按行执行”的误区
•   不是真·逐行读。Python 加载时会先全文件编译成字节码，遇到语法错误直接崩，然后再由 VM 执行。

5. JIT 的边界模糊
•   Java/JS (V8)：解释执行 + 热点代码 JIT 编译（变机器码）。

•   CPython：基本纯解释，极少 JIT。

一句话总结：
解释器不是为了翻译而存在，而是为了支撑“跨平台”和“动态语言特性”而存在的运行时环境。
