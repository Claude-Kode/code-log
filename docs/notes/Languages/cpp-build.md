  ### 1. 合并 (Merging)
  链接器将多个目标文件(.o)及库文件(.a)中的同类段(Section)进行拼接。
  - 将所有 .text (代码段) 合并为一个统一的代码区。
  - 将所有 .data (初始化数据段) 和 .bss (未初始化数据段) 分别合并。
  目的是将分散的数据组织成规整的可执行文件布局。

  ### 2. 符号解析 (Symbol Resolution)
  检查目标文件中的“未定义符号”(如调用的函数名)，并在其他目标文件或静态库的“导出符号表”中寻找定义。
  - 若找到匹配项，记录引用关系。
  - 若未找到，抛出 `undefined reference` 错误并终止链接。
  确保每个全局符号都有唯一的定义实体。

  ### 3. 地址重定位 (Relocation)
  为合并后的代码和数据分配最终的虚拟内存地址。
  - 编译阶段生成的指令使用的是临时占位地址。
  - 链接器确定各函数在代码段中的偏移量后，修改指令中的地址字段，将占位符替换为实际的运行时地址。
  确保 CPU 执行时能跳转到正确的函数入口。

  ### 最终结果
  生成一个包含启动代码、用户代码、库代码及数据的单一 ELF/Mach-O/PE 格式可执行文件。 
  最后生成的 exe 文件是完全自依赖的

### 补充关键点

**1. 编译差异 (-fPIC)**
生成动态库前必须加 `-fPIC`（Position-Independent Code，位置无关代码）。
因为动态库在内存中的加载地址是不固定的，`-fPIC` 确保代码无论被加载到内存何处都能正常执行，而静态库则无此强制要求。

**2. 符号重定位机制 (GOT/PLT)**
动态链接器并非简单地“填空”。它维护两张表：
*   **GOT (全局偏移表)**：存储全局变量/函数的实际地址。
*   **PLT (过程链接表)**：存储跳转到 GOT 中地址的代码片段。
首次调用 `add` 时，通过 PLT 触发动态链接器查找 `libmath.so` 中的 `add` 地址，填入 GOT；后续调用直接通过 GOT 跳转，实现懒加载（Lazy Binding）。

**3. 内存共享优势**
若 10 个进程运行 `app_dynamic`，内存中只需加载一份 `libmath.so` 的代码段（只读），仅数据段（可写）为每个进程私有。这极大节省了内存和磁盘空间。

**4. 依赖验证**
链接完成后，使用 `ldd app_dynamic` 可查看程序依赖的动态库列表。若运行时提示 "cannot open shared object file"，需通过 `LD_LIBRARY_PATH` 环境变量或 `/etc/ld.so.conf` 配置库路径。

### 补充内容
#### 动态库文件形式
跨平台后缀不同：Linux为`.so`（Shared Object），Windows为`.dll`（Dynamic Link Library），macOS为`.dylib`。本质是带动态链接段的ELF（Linux）/PE（Windows）格式二进制文件，结构与可执行文件类似，但无独立程序入口，仅包含可被加载的代码段与符号表。

### 静态库文件形式
静态库本质是**归档文件（Archive）**，Linux/Unix下后缀为`.a`，Windows下为`.lib`（静态版）。
- **内部结构**：由多个`.o`（目标文件）打包而成，使用`ar`（archive）工具创建。文件头包含魔数标识（`!<arch>`），随后是每个`.o`文件的元数据索引（文件名、时间戳、权限等）和原始二进制数据。
- **关键特征**：包含**符号索引表**（由`ranlib`或`ar s`生成），链接器通过该表快速定位所需符号位于哪个`.o`文件中，避免逐文件扫描。
- **格式对比**：不同于动态库/可执行文件的ELF/PE格式，静态库本身不是可执行格式，仅是目标文件的“集装箱”。可用`ar -t libxxx.a`查看包含的目标文件列表，用`objdump -t`查看符号索引。

#### 动静态库优缺点对比
| 维度 | 静态库 | 动态库 |
| --- | --- | --- |
| 运行时依赖 | 无，可独立运行 | 必须存在对应库文件，缺失则启动失败 |
| 文件体积 | 较大，含全部依赖代码 | 较小，仅存符号引用 |
| 更新成本 | 库改动需重新编译程序 | 库改动仅需替换库文件（接口兼容前提） |
| 内存占用 | 多进程运行时重复加载相同代码 | 多进程共享同一份内存中的库代码 |
| 启动速度 | 快，无额外加载步骤 | 略慢，需运行时加载库并完成重定位 |
| 适用场景 | 小型程序、运行环境不可控场景 | 公共组件、插件系统、高频更新模块 |
