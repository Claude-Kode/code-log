# 引言
本人为在校计算机相关专业学生，长期深耕竞赛C++与算法领域，拥有市级、全国程序设计竞赛获奖经历，在校以编程、算法学习为核心主线。当前自身代码能力偏向算法竞赛刷题，擅长单文件逻辑实现，但缺乏工业级软件工程项目开发经验。
撰写本职业生涯规划书，旨在梳理自身能力与行业岗位的差距，明确长期发展赛道为C++游戏引擎底层开发工程师，制定可落地的阶段性学习、实战计划，弥补工程开发短板，稳步完成从竞赛选手到游戏底层研发工程师的转型，最终实现独立研发商用游戏引擎核心模块的长期职业目标。

# 一、自我认知
## 1.1 职业兴趣
我对计算机底层程序运行逻辑、高性能计算、实时图形渲染抱有强烈兴趣，相较于上层业务开发，更专注底层框架、性能优化、内存管理、渲染管线等硬核技术；热衷于C++语言长期深耕，喜欢拆解开源引擎源码、自主搭建小型渲染项目，享受通过代码优化程序运行效率、从零搭建软件框架的过程，明确倾向游戏引擎底层研发方向。

## 1.2 职业能力
1. 算法与C++基础能力突出：熟练各类常用数据结构、图论、分块、可持久化数据结构，掌握STL全系列容器使用，精通位运算、本地调试、对拍测试、文件IO等竞赛工具链；
2. 竞赛成果背书：重庆市程序设计竞赛金奖、CCPC南昌全国邀请赛铜奖，具备较强逻辑推导、复杂问题拆解、极限性能优化思维；
3. 现有短板：仅掌握单文件编程，无多文件工程、CMake、编译链接、内存模型、图形API、大型项目架构设计相关经验。

## 1.3 职业价值观
1. 技术长期深耕：优先选择可以持续沉淀底层技术、不存在快速技术迭代淘汰的岗位，拒绝短期表层业务开发；
2. 技术价值导向：看重岗位对程序性能、底层架构的打磨空间，追求代码质量与程序运行效率；
3. 成长优先：择业时优先考虑能够提供源码研读、底层研发实践、资深工程师指导的研发平台；
4. 稳定发展：希望长期深耕游戏引擎赛道，不频繁更换技术栈，建立完整底层技术知识体系。

## 1.4 个人特质
性格耐心专注，能够长期投入长时间源码阅读、复杂算法调试；做事逻辑条理清晰，习惯整理模板、文档梳理知识点；自学能力较强，擅长自主拆分复杂学习任务；做事严谨，注重程序边界条件、稳定性与性能；缺点是工程协作、大型项目设计经验不足，对图形学理论理解入门较慢。

## 1.5 他人眼中的我
1. 任课老师：算法思维突出，代码实现速度快，逻辑严谨，但项目工程化意识薄弱，缺少大型软件项目实践；
2. 竞赛队友：擅长性能优化、复杂数据结构落地，独立刷题能力强，但不擅长多人协作拆分大型项目；
3. 身边同学：自律性强，长期坚持编程学习，愿意主动钻研冷门底层技术，擅长整理学习笔记与工具模板。

## 1.6 自我认知小结
本人核心优势为算法逻辑、竞赛C++代码功底、自主学习能力，天然适配游戏引擎高性能底层开发岗位；核心短板集中在工业工程开发、图形学知识、大型项目架构设计。个人兴趣、价值追求均高度匹配游戏引擎底层研发赛道，具备转型发展的基础条件，只需针对性补齐工程、图形相关短板即可向目标岗位靠拢。

# 二、职业认知
## 2.1 家庭环境
家人支持计算机技术长期发展路线，无强制择业限制，经济条件能够支撑自主购买学习设备、线上技术课程；家庭无行业相关从业者，无法提供行业内推、经验指导，但家人尊重我的技术选择，愿意给予充足时间完成在校阶段技术积累，不要求过早就业。

## 2.2 学校环境
本校计算机专业开设C/C++、数据结构、计算机图形学、操作系统核心课程，具备程序设计竞赛训练实验室，拥有稳定的算法学习氛围；校内缺少游戏引擎专项实训项目，无游戏研发校企合作实训渠道，图形学课程教学内容偏理论，缺少工程落地实操训练，需要依靠课外自主学习补齐行业实战内容。

## 2.3 社会环境
1. 就业政策：数字文创、游戏产业为国家数字经济扶持行业，国产自研引擎（如Unity中国版、UE国产化、各类独立自研引擎）政策扶持力度提升，底层研发人才缺口扩大；
2. 就业形势：互联网上层业务岗位竞争内卷严重，但游戏、渲染、引擎底层C++工程师供给不足，中高端底层研发人才长期紧缺；
3. 行业竞争对手：竞争者分为计算机图形学专业科班学生、有游戏实习经历的开发、计算机竞赛转底层研发选手，科班学生图形基础更强，竞赛选手算法性能优化能力更强。

# 三、职业分析
## 3.1 行业分析（游戏引擎底层研发行业）
1. 行业现状：国内游戏产业稳步发展，游戏厂商逐步摆脱海外商业引擎依赖，大力投入自研游戏引擎开发，移动端、主机、云游戏多终端渲染需求爆发；
2. 发展趋势：实时光线追踪、并行多线程渲染、ECS架构、轻量化跨平台引擎为未来核心发展方向，高性能、低功耗底层模块人才需求持续上涨；
3. 人才需求：企业急需掌握C++、图形API、内存优化、引擎架构的底层开发工程师，偏好具备算法优化能力、能独立维护核心模块的求职者；
4. 人行业匹配：自身竞赛算法、C++基础完全契合行业性能优化需求，仅补充图形、工程知识即可匹配行业人才标准，匹配度较高。

## 3.2 职业分析（C++游戏引擎开发工程师）
1. 工作内容：引擎底层框架开发、渲染管线优化、内存池/资源管理模块编写、多线程并行计算、引擎性能瓶颈排查、跨平台适配、基础物理与碰撞模块研发；
2. 工作要求：精通标准C++、熟悉编译构建工具、掌握OpenGL/Vulkan图形API、了解计算机图形学基础、具备大型项目调试与优化能力；
3. 发展前景：初级底层工程师→引擎核心开发→渲染技术专家→引擎架构师，可横向拓展图形渲染、主机开发、云游戏技术路线；
4. 生活方式：偏向技术研发岗，以源码开发、性能调试为主，项目上线周期内会阶段性加班，日常工作专注底层技术深耕，符合个人长期技术钻研需求。

## 3.3 单位分析
目标单位分为三类：
1. 头部游戏大厂自研引擎部门：米哈游、网易、腾讯光子等自研引擎团队，完善技术培养体系，项目体量庞大，资深技术专家多；企业文化重视技术自研，研发氛围浓厚；
2. 专业引擎研发企业：专注商用引擎开发公司，深耕渲染底层，岗位技术专一；
3. 中小型独立游戏工作室自研团队：项目体量小，能够全程参与引擎从零搭建，综合锻炼引擎全模块能力。
本人中长期目标优先冲刺头部大厂自研引擎团队，匹配自身长期深耕底层的职业追求。

## 3.4 职位分析（初级C++游戏引擎开发工程师）
1. 岗位职责：协助维护引擎底层基础模块、负责基础渲染功能迭代、定位程序内存泄漏与性能瓶颈、编写单元测试、适配多平台底层接口；
2. 岗位胜任素质：扎实C++17/20基础、理解编译链接与工程构建、基础图形学渲染知识、具备代码优化思维、能阅读大型开源项目源码、良好文档编写能力；
3. 人岗匹配：当前仅满足C++算法优化相关素质，工程、图形、项目源码阅读能力存在明显缺口，为在校阶段核心补齐方向。

## 3.5 地域分析
优先发展城市：上海、成都、广州、深圳；
1. 城市发展前景：上述城市聚集国内头部游戏厂商、自研引擎工作室，产业配套完善，底层研发岗位供给充足；
2. 文化与生活：互联网技术氛围浓厚，程序员群体规模大，技术交流渠道丰富；
3. 人际发展：聚集大量图形、引擎行业从业者，便于实习、内推、技术交流，适配游戏引擎研发长期发展。

# 四、职业定位
## 4.1 职业发展前景
游戏国产化浪潮下自研引擎赛道长期向上，底层C++渲染研发人才稀缺，岗位薪资、晋升空间优于普通应用开发；技术具备强不可替代性，工作年限越长技术沉淀价值越高，中年职业危机更低；可延伸至车载渲染、影视实时渲染、云计算图形等多赛道，职业选择广度充足。

## 4.2 职业风险
1. 技术迭代风险：图形API、硬件渲染标准持续更新，需要终身持续学习新技术；
2. 行业周期风险：游戏行业存在版号、市场周期波动，中小厂商存在业务收缩裁员可能；
3. 能力门槛风险：底层研发学习曲线陡峭，图形学理论难度高，长期学习投入量大；
4. 竞争风险：科班图形专业、海外留学生持续涌入赛道，高端岗位竞争激烈。

## 4.3 职业要求
1. 硬技能要求：精通现代C++、CMake工程构建、操作系统内存/线程、Vulkan/OpenGL图形API、计算机图形学、数据结构与高性能算法；
2. 软素质要求：严谨的工程思维、源码阅读理解能力、性能排查思维、文档撰写、基础团队协作能力；
3. 学历与实践要求：计算机相关专业，具备个人引擎/渲染项目作品集，大厂实习经历加分。

# 五、职业决策
## 5.1 个人SWOT分析
### 5.1.1 优势（Strength）
1. C++算法功底扎实，拥有市级、全国程序设计竞赛奖项，擅长程序性能、复杂度优化；
2. 长期自主学习编程，自律性强，擅长整理技术模板、梳理完整知识体系；
3. 熟练掌握算法竞赛全套调试工具，具备较强bug排查、对拍测试思维；
4. 个人兴趣与目标岗位高度统一，愿意长期投入底层技术深耕，无频繁换赛道想法。

### 5.1.2 劣势（Weakness）
1. 仅会单文件竞赛代码，无多文件大型C++工程项目开发经验，不熟悉CMake、编译链接流程；
2. 计算机图形学、图形API零基础，缺少渲染相关理论与实操；
3. 无企业实习、团队协作开发经历，不了解工业代码规范、单元测试流程；
4. 对内存模型、RAII、多线程、模板元编程等现代C++工程内容理解浅薄。

### 5.1.3 机遇（Opportunity）
1. 国内扶持自研游戏引擎，底层C++研发人才缺口持续扩大；
2. 开源引擎、渲染教程资源丰富，线上可低成本完成自学与项目实战；
3. 在校阶段时间充裕，可完整规划2年以上长期学习路线，提前积累作品集；
4. 算法竞赛经历是企业筛选底层性能开发人才的加分项，区别于普通求职者。

### 5.1.4 威胁（Threat）
1. 计算机图形学科班学生系统学习渲染理论，专业基础优于我；
2. 大量计算机专业学生涌入游戏开发赛道，基础开发岗位竞争激烈；
3. 图形硬件、渲染技术更新速度快，需要持续跟进学习，无法停滞；
4. 游戏行业受政策、市场环境影响，存在岗位收缩、裁员周期性风险。

## 5.2 SMART原则制定职业目标
### 5.2.1 具体（Specific）
长期职业目标：成为C++游戏引擎底层研发工程师，能够独立设计、开发商用游戏引擎渲染、内存管理核心模块，未来向引擎架构师发展；
在校阶段具体目标：补齐C++工程能力、掌握计算机图形基础、完成自研迷你2D引擎作品集，毕业前拿到头部游戏厂商引擎部门实习offer。

### 5.2.2 可衡量（Measurable）
1. 短期1个月：独立使用CMake搭建多文件C++项目，自主编写通用调试头文件；
2. 中期6个月：掌握基础Vulkan渲染流程，完成可运行2D渲染Demo；
3. 长期在校结束前：完整自研迷你2D引擎，具备资源加载、渲染、碰撞检测完整模块，产出完整开源作品集。

### 5.2.3 可实现（Attainable）
目标匹配自身竞赛C++基础，拆分三阶段循序渐进学习，不存在跨越式不切实际的要求；所有学习内容均有成熟线上教程、开源项目可供参考，在校充足课余时间可支撑计划落地；不脱离自身算法优势，优先强化擅长的性能优化模块，降低转型难度。

### 5.2.4 与时间相关（Time-bound）
1. 1～2个月：完成C++工程化基础补齐；
2. 3～6个月：图形学基础+引擎架构理论学习，产出简易渲染Demo；
3. 6个月至毕业：持续自研引擎项目、研读开源引擎源码、投递实习。

### 5.2.5 与现实相关（Relevant）
贴合游戏引擎行业人才招聘真实需求，充分利用自身竞赛算法优势，弥补在校缺少实训的短板；兼顾家庭、学业现实情况，不脱离计算机专业主线，长期赛道稳定，就业前景可观。

# 六、分阶段行动实施计划
## 6.1 阶段1：补齐C++工程化基础（1～2个月）
1. 系统吃透预处理、编译、汇编、链接完整流程，掌握头文件保护、循环依赖、符号未定义报错底层原理；
2. 熟练拆分.h/.cpp多文件项目，独立编写CMakeLists.txt管理工程，通过宏LOCAL统一管理本地调试环境；
3. 系统学习操作系统内存模型、智能指针、移动语义、const常量语义，掌握RAII资源自动管理思想；
4. 深入理解模板底层机制：模板实例化、类/函数模板特化、if constexpr编译期分支，能够自主改造泛型工具，不再单纯复制模板代码。

## 6.2 阶段2：游戏引擎前置知识储备（3～6个月）
1. 系统学习计算机图形学基础，入门OpenGL、Vulkan图形API，吃透基础渲染流水线、坐标变换、纹理绘制逻辑；
2. 梳理完整商用游戏引擎架构：资源管理系统、渲染循环、物理碰撞模块、输入系统、引擎配套工具链；
3. 结合自身算法优势，专项学习引擎高性能优化方案：内存池、空间划分树、批渲染、并行多线程计算；
4. 小型实战落地：搭建简易2D轻量渲染模块，实现资源加载、纹理绘制、帧率控制、基础碰撞检测完整运行流程。
